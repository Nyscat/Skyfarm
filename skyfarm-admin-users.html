How it works
  • Sending an invite uses sendSignInLinkToEmail. When the user clicks the link and signs in, Firebase creates their Auth account.
  • We also store a record in Firestore → users/{email} with role and allowedProperties. Your app pages can read this for authorization.
  • Listing/Deleting here affects the Firestore user record only. Listing Auth users or creating passwords requires the Admin SDK (server).
